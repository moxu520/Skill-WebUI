# Skill-WebUI

[English](./README.md)

Skill-WebUI 是一个面向本地文件系统的技能工作台，用来管理基于 Markdown 的技能目录。它提供了围绕技能浏览、新建、编辑、导入、发现和翻译的一套 Web UI，同时保留了足够直接的目录结构，便于继续沿用普通仓库和文件管理流程。

项目基于 Next.js、React、TypeScript 构建，数据层直接读写本地技能目录。当前版本更适合个人开发者或小团队在本地维护技能库，而不是把它当作托管式在线服务。

## 功能亮点

- 基于目录和 `SKILL.md` 的本地优先技能工作区
- 在界面中直接新建、编辑、重命名和删除受管技能
- 支持从本地路径导入技能
- 支持从 Git 仓库发现并导入技能
- 自动扫描常见本地目录，并支持配置额外扫描根目录
- 支持中英文双向翻译受管技能
- 提供独立设置中心，用于工作区、技能发现和 provider 相关配置

## 当前能力

### 技能工作区

- 以网格或列表方式浏览受管技能
- 按名称或更新时间搜索、排序技能
- 在应用内预览 Markdown 内容
- 编辑标题、描述、正文内容以及目录标识
- 从工作区删除受管技能目录

### 导入与发现

- 从本地目录直接导入技能
- 扫描已配置的本地根目录，发现可导入技能
- 输入 Git 仓库地址后发现候选技能并按需导入
- 在导入前识别冲突项和无效候选项

### 翻译

- 选择已有受管技能并生成翻译结果
- 支持英文到中文、中文到英文两种方向
- 可选择覆盖当前技能，或另存为派生副本
- 当前可执行的 provider：
  - `Google`
  - `local_builtin`

### 设置

- 查看当前受管技能根目录
- 查看技能发现配置文件路径
- 配置额外扫描根目录
- 查看翻译 provider 的当前接入状态
- 为后续模型服务接入预留统一入口

## 快速开始

### 环境要求

- 推荐 Node.js 20+
- npm

### 安装

```bash
git clone https://github.com/moxu520/Skill-WebUI.git
cd Skill-WebUI
npm install
```

### 本地开发

```bash
npm run dev
```

浏览器访问 `http://localhost:3000`。

### 构建与检查

```bash
npm run build
npm run lint
```

## 配置说明

### `SKILLS_ROOT`

默认受管技能根目录为：

```text
<项目根目录>/skills
```

可以通过 `SKILLS_ROOT` 环境变量覆盖。

示例：

```bash
SKILLS_ROOT=/absolute/path/to/skills npm run dev
```

### 技能发现配置

额外扫描根目录会持久化到应用维护的本地 JSON 配置文件中。设置页还会展示：

- 当前受管技能根目录
- 技能发现配置文件路径
- 可编辑的额外扫描目录

## 典型使用流程

1. 使用 `npm run dev` 启动项目。
2. 打开 Skills 工作区。
3. 新建技能，或者从本地路径 / Git 仓库导入技能。
4. 检查并编辑生成后的 `SKILL.md` 内容。
5. 需要中英文版本时，进入 Translation 工作区处理。
6. 如果技能目录不在默认扫描位置，到 Settings 中补充扫描根目录。

## 当前限制

- 当前定位仍是本地工作区，不是多用户在线平台。
- 翻译 provider 的接入范围在当前实现中还比较有限。
- 模型服务设置页目前主要是为后续接入预留入口。
- 有些侧边导航已经展示为规划中的能力，但尚未提供完整功能。
